# Googleドライブ自動保存：セットアップ手順（GAS方式）

録音音声を社長のGoogleドライブへ自動保存するための設定です。
**Google Apps Script（GAS）** の小さなウェブアプリを経由する方式で、
Google Cloud Console も OAuth も不要。所要 5分・無料・一度設定すれば切れません。

> 仕組み：アプリが音声をGASに送る → GASが「社長本人として」社長のドライブに保存。
> 保存先は `（保存先フォルダ）/（受験者_日付）/A-1_母豚の健康観察.webm` の形。

---

## 手順（全部で3ステップ）

### ステップ1：GASプロジェクトを作ってコードを貼る
1. https://script.google.com/ を開く（社長のGoogleでログイン）
2. 左上「**＋ 新しいプロジェクト**」
3. 表示されたコード（`function myFunction()…`）を**全部消して**、
   このリポジトリの **`gas/Code.gs` の中身を全部貼り付け**
4. コード上部の
   ```
   var TOKEN = 'oral-exam-2026-CHANGE-ME';
   ```
   の `oral-exam-2026-CHANGE-ME` を、**自分で決めた合言葉**に書き換える
   （例：`var TOKEN = 'ooiri-koutou-2026';`）。この合言葉は後でアプリにも入れます。
5. 💾 保存（Ctrl+S）

### ステップ2：ウェブアプリとしてデプロイ
1. 右上「**デプロイ**」→「**新しいデプロイ**」
2. 歯車（種類の選択）→「**ウェブアプリ**」
3. 設定：
   - 説明：任意（例 `oral-exam drive`）
   - **次のユーザーとして実行：自分**
   - **アクセスできるユーザー：全員**
4. 「**デプロイ**」→ 初回は権限の確認が出る：
   - 「アクセスを承認」→ 自分のアカウントを選択
   - 「このアプリは確認されていません」→ **詳細 →（プロジェクト名）に移動** → **許可**
     （自分が作ったスクリプトなので問題ありません）
5. 表示された「**ウェブアプリ**」の **URL**（`https://script.google.com/macros/s/……/exec`）を**コピー**

> あとで貼り直したいとき：「デプロイ」→「デプロイを管理」からURLを確認できます。
> コードを直したら「デプロイを管理 → 編集（鉛筆）→ バージョン＝新バージョン → デプロイ」で反映。

### ステップ3：アプリに貼って接続テスト
本番アプリ → **設定タブ →「Googleドライブに自動保存」**
1. **ウェブアプリURL（GAS）**：ステップ2でコピーしたURLを貼る
2. **合言葉**：ステップ1で決めた値（GASの `TOKEN` と**同じ値**）を入れる
3. **保存先フォルダ名**：そのままでOK（既定 `口頭試問音声`）
4. 「**設定を保存**」→「**接続テスト**」
   - 「接続OK：ドライブに保存できます」と出れば成功 ✅
5. 「**録音を自動でドライブに保存する**」に**チェック**

これで完了。録音するたびに、社長のドライブの
`口頭試問音声/受験者_日付/A-1_母豚の健康観察.webm`
に自動保存されます（各録音の下に「☁ ドライブ保存済み」と表示）。

---

## 使い方（保存後）
- PCでGoogleドライブを開く → 上記フォルダに音声が揃っている
- ダウンロードして、お手持ちの文字起こしツール（Whisper等）に通す

## よくあるつまづき
- **接続テストが失敗する**
  - URLが `…/exec` で終わっているか（`…/dev` ではない）
  - アプリの合言葉とGASの `TOKEN` が**完全一致**しているか
  - デプロイの「アクセスできるユーザー」が **全員** になっているか
- **コードを直したのに反映されない** → デプロイは固定URL。再デプロイ時に「**新バージョン**」を選ぶ
- **スマホでも使う** → 同じ本番URLをスマホで開き、設定に同じURL＋合言葉を入れるだけ（追加設定不要）

## セキュリティの注意
- 「アクセスできるユーザー：全員」は、URLを知っていれば誰でもPOSTできる状態です。
  そのため**合言葉（TOKEN）でガード**しています。合言葉は推測されにくい値にし、URLと合言葉を不用意に共有しないでください。
- 合言葉を変えたいときは、GASの `TOKEN` を変更して**再デプロイ（新バージョン）**し、アプリ側の合言葉も同じ値に更新します。
