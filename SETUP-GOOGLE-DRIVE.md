# Googleドライブ自動保存：セットアップ手順

録音音声を社長のGoogleドライブへ自動保存するための **1回だけの設定** です。
所要 5〜10分・無料。難しい用語は読み飛ばしてOK、画面の通りに進めれば終わります。

> ポイント：このアプリが使う権限は **`drive.file`（このアプリが作ったファイルだけ）**。
> 社長のドライブ全体を見ることはありません。だからGoogleの面倒な審査も不要です。

---

## 事前メモ：アプリのURL（オリジン）を決める
Googleは「どのURLからのアクセスを許可するか」を登録する必要があります。
- **ローカルでお試し**：`http://localhost:8765`
- **本番公開後**：例 `https://shoichiokada225-sys.github.io`（GitHub Pagesの場合。末尾の `/...` は付けず、ドメインまで）

この2つを後の手順6で登録します（両方入れておけば、お試しも本番も動きます）。

---

## 手順

### 1. Google Cloud Console を開く
https://console.cloud.google.com/ にGoogleアカウントでログイン。

### 2. プロジェクトを作成
画面上部のプロジェクト選択 →「新しいプロジェクト」→ 名前（例：`oral-exam`）→ 作成 → 作ったプロジェクトを選択。

### 3. Google Drive API を有効化
左メニュー「APIとサービス」→「ライブラリ」→ `Google Drive API` を検索 →「有効にする」。

### 4. OAuth 同意画面を設定
「APIとサービス」→「OAuth同意画面」
- User Type：**外部**（External）→ 作成
- アプリ名：`口頭試問 評価システム`／ユーザーサポートメール：自分のメール／デベロッパー連絡先：自分のメール → 保存して次へ
- スコープ：そのまま「保存して次へ」（後でアプリ側がdrive.fileを要求します）
- **テストユーザー**：「+ ADD USERS」で **自分のGmailアドレスを追加**（重要）→ 保存して次へ
  - ※「公開ステータス＝テスト中」のままでOK。テストユーザーに入れた自分のアカウントなら使えます。

### 5. 認証情報（OAuthクライアントID）を作成
「APIとサービス」→「認証情報」→「+ 認証情報を作成」→「OAuth クライアント ID」
- アプリケーションの種類：**ウェブアプリケーション**
- 名前：`oral-exam-web`

### 6. 承認済みの JavaScript 生成元（オリジン）を登録
「承認済みのJavaScript生成元」に **＋ URIを追加** で以下を入れる：
- `http://localhost:8765` （お試し用）
- （本番公開後）公開URLのドメイン。例 `https://shoichiokada225-sys.github.io`

> 「承認済みのリダイレクトURI」は **空のままでOK**（このアプリはトークン方式で使います）。

→「作成」。

### 7. クライアントIDをコピー
表示される **クライアントID**（`xxded.apps.googleusercontent.com` のような文字列）をコピー。
> 「クライアントシークレット」は使いません（このアプリには入力不要）。

### 8. アプリに貼り付けて接続
アプリの **設定タブ →「Googleドライブに自動保存」**
1. 「OAuth クライアントID」に貼り付け → 「設定を保存」
2. 「Googleに接続」→ Googleのログイン画面が出る
   - 「このアプリは確認されていません」と出たら → **詳細 → （安全でないページに移動）** を選択（自分が作ったアプリなので問題ありません）
   - ドライブへのアクセスを **許可**
3. 「● 接続済み」になったら →「録音を自動でドライブに保存する」に **チェック**

これで完了。以後、録音するたびに
`（保存先フォルダ）/（受験者_日付）/A-1_母豚の健康観察.webm`
の形でドライブに自動保存されます。

---

## 使い方（保存後）
- PCでGoogleドライブを開く → 上記フォルダに音声が揃っている
- ダウンロードして、お手持ちの文字起こしツール（Whisper等）に通す

## よくあるつまづき
- **「接続に失敗」**：手順6のオリジンが、今アプリを開いているURLと一致しているか確認（`http://localhost:8765` で開く。`127.0.0.1` で開いた場合は `http://127.0.0.1:8765` も登録するか、`localhost` で開き直す）。
- **ログイン画面が出ない／ライブラリ読み込み中**：少し待ってから再度「Googleに接続」。
- **トークンの有効期限**：接続は約1時間で切れます。切れたら「Googleに接続」を押し直すだけ。
- **スマホでも使う**：本番URL（手順6の本番オリジン）を登録し、スマホのブラウザでそのURLを開いて同じく接続。
